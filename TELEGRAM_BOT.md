# Telegram Bot - Руководство пользователя

## Описание

Telegram бот для управления FunPay воркерами. Позволяет регистрироваться, добавлять токен FunPay и управлять автоматической обработкой сообщений.

## Возможности

- 🔐 Регистрация с сохранением FunPay токена
- ▶️ Запуск воркера для автоматической обработки сообщений
- ⏸️ Остановка воркера
- 📊 Проверка статуса воркера
- 🔔 Уведомления о новых заказах

## Команды

### `/start`
Приветственное сообщение и краткое описание бота.

### `/help`
Показывает список всех доступных команд.

### `/register`
Начинает процесс регистрации. Бот попросит вас отправить ваш `golden_token` от FunPay.

**Как получить токен:**
1. Войдите на FunPay в браузере
2. Откройте DevTools (F12)
3. Перейдите в Application → Cookies
4. Найдите cookie с именем `golden_key`
5. Скопируйте его значение

**Безопасность:**
- Токен сохраняется в базе данных
- Сообщение с токеном автоматически удаляется после отправки
- Токен используется только для работы с вашим аккаунтом FunPay

### `/start_worker`
Запускает воркера для вашего аккаунта. Воркер будет:
- Получать новые сообщения от клиентов
- Отправлять их в AI для генерации ответов
- Автоматически отвечать клиентам
- Уведомлять вас о новых заказах

**Требования:**
- Вы должны быть зарегистрированы (`/register`)
- Токен должен быть валидным

### `/stop_worker`
Останавливает воркера. Автоматическая обработка сообщений прекращается.

### `/status`
Показывает текущий статус:
- Запущен ли воркер
- Ваш Telegram ID
- Статус регистрации

### `/cancel`
Отменяет текущее действие (например, процесс регистрации).

## Пример использования

1. **Регистрация:**
   ```
   Вы: /register
   Бот: 🔑 Регистрация
        Отправьте ваш golden_token от FunPay...
   
   Вы: [отправляете токен]
   Бот: ✅ Регистрация успешна!
   ```

2. **Запуск воркера:**
   ```
   Вы: /start_worker
   Бот: ✅ Воркер запущен!
        Бот начал обрабатывать сообщения с FunPay.
   ```

3. **Получение уведомления о заказе:**
   ```
   Бот: 🔔 Новый заказ #12345!
        Пожалуйста, зайдите на FunPay для обработки заказа.
   ```

4. **Остановка воркера:**
   ```
   Вы: /stop_worker
   Бот: ✅ Воркер остановлен!
   ```

## Запуск бота

### Настройка

1. Создайте бота через [@BotFather](https://t.me/BotFather)
2. Получите токен бота
3. Добавьте токен в `.env`:
   ```env
   BOT_TOKEN=your_telegram_bot_token_here
   ```

### Запуск

```bash
# Запустить Telegram бота
poetry run python src/run_telegram_bot.py

# Запустить FastAPI сервер (в отдельном терминале)
poetry run uvicorn src.app:app --reload
```

**Важно:** Оба процесса должны быть запущены одновременно:
- Telegram бот - для взаимодействия с пользователями
- FastAPI сервер - для работы воркеров и обработки событий

## Архитектура

```
Пользователь (Telegram)
    ↓

    ↓
Database (сохранение токена)
    ↓

    ↓
FunpayWorker (обработка событий)
    ↓

    ↓
FunPay (отправка ответов)
```

- ✅ Каждый пользователь управляет только своим воркером

## Troubleshooting

### Бот не отвечает
- Проверьте, что `BOT_TOKEN` указан в `.env`
- Убедитесь, что бот запущен (`python src/run_telegram_bot.py`)
- Проверьте логи на наличие ошибок

### Воркер не запускается
- Убедитесь, что FastAPI сервер запущен
- Проверьте валидность токена FunPay
- Проверьте логи воркера

### Не приходят уведомления о заказах
- Убедитесь, что воркер запущен (`/status`)
- Проверьте, что Telegram бот имеет права отправлять сообщения
- Проверьте логи на наличие ошибок

## Логирование

Все действия логируются:
- Регистрация пользователей
- Запуск/остановка воркеров
- Ошибки при отправке уведомлений

Логи помогают отслеживать работу системы и диагностировать проблемы.
